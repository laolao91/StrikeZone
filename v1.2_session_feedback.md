# StrikeZone — Session Feedback (2026-06-27)

## v1.2 intentions
- Rebuild on the new SDK
- Speed improvements: phone poll reduced to 3s, pending-render flag so mid-cascade updates aren't dropped
- UI/UX improvements (see below)
- Fix confirmed bugs from code review


## User-reported issues

1. **App defaults to old game on launch** (e.g. "STL 9-6 MIN Final") — `game-state.ts:83` calls `refresh()` for any persisted `selectedGamePk` without checking if it exists in today's schedule.

2. **Glasses scroll UX broken** — Must scroll to the bottom of the screen before cursor moves. Root cause: `gameLabel()` for Preview games appends `"1:10 PM"` as one string; the SDK word-wraps on the space before "PM", so each game takes 2 lines. 8+ games × 2 lines overflows the text container.

3. **Phone view should show the date** — `SettingsApp.tsx` shows today's games but no date in the header.

4. **Favorite teams** — Requested: allow favoriting teams so they sort to top of game list. `AppSettings` currently only holds `selectedGamePk`.

5. **Location/proximity** — User wants the app to detect they're at a ballpark (e.g. Citi Field) and surface that game first. No geolocation code exists.

6. **Strikezone render is working** ✅ — User confirmed the latest version's strikezone ASCII/text render approach is "good and working." Text-zone approach is approved; image cascade approach is still unconfirmed on hardware.

## Navigation pattern reference

User wants game navigation similar to SubwayLens's launch menu / selector. See `SubwayLens_v2.0.0/` for the pattern (launch menu, nearest-first sorting).
